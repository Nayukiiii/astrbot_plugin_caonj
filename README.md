# astrbot_plugin_cao群友

AstrBot QQ群插件，下载即可开箱使用，把需要草的群友换成自己想要的人选就行！

> 此仓库由 [Claude](https://claude.ai) 编写

---

## 功能

- **草群友** — 向群友发起草草，成功/失败基于密码学真随机判定
- **群友反草** — 草群友失败时群友有概率反草，并要求对方选择里面/外面
- **群友体内液体** — 查看群友体内累计注入量及排行榜
- **群友战绩** — 查看群友近30天反草战绩排行（注入总量榜 + 被注入次数榜）
- **杂鱼排行** — 查看近30天选择外面的杂鱼排行（放弃次数榜 + 放弃总量榜）
- **草群友排行** — 查看群内草群友排行
- **草群友关系图** — 可视化群内草群友关系网络
- **重置草nj** — 重置今日草nj次数（普通用户需掷骰，bot管理员直接重置）

---

## 安装

1. 在 AstrBot 插件管理中上传或克隆本仓库
2. 在插件配置中填入 `**_qq`（群友的QQ号）和 `**_name`（群友的名字）
3. 重启 AstrBot，即可使用

## 自定义配置

### 插件配置项

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `群友_qq` | 群友的QQ号 | — |
| `群友_name` | 群友的名字 | 宁隽 |
| `caonj_probability` | 草群友触发概率（%） | 30 |
| `fancao_probability` | 反草基础强度（%），设0则关闭反草 | 50 |
| `caonj_daily_limit` | 每人每天草群友次数上限 | 5 |
| `reset_daily_attempts` | 普通用户每天重置尝试次数上限 | 3 |

### 称号与提示词

插件目录下有六个 JSON 文件，可自行编辑：

| 文件 | 说明 |
|------|------|
| `群友_body_titles.json` | 群友体内排行图的称号标签（按累计量/次数） |
| `群友_battle_titles.json` | 群友战绩排行图的称号标签（按累计量/次数） |
| `outside_titles.json` | 杂鱼排行图的称号标签（按放弃量/次数） |
| `群友_body_comments.json` | 草群友选里面后的提示词（按本次注入量） |
| `群友_battle_comments.json` | 反草选里面后的提示词（按本次注入量） |
| `nj_fancao_comments.json` | 反草成功时的台词（按仇恨值分档） |

每个区间可以写多条，每次随机抽一条显示。示例：

```json
{
  "ml_tiers": [
    {"min_ml": 4000, "comments": ["文案A", "文案B"]},
    {"min_ml": 0,    "comments": ["文案C"]}
  ]
}
```

反草台词按仇恨值分档：

```json
{
  "grudge_tiers": [
    {"min_grudge": 0.8, "comments": ["老仇人文案"]},
    {"min_grudge": 0.4, "comments": ["中等仇恨文案"]},
    {"min_grudge": 0.0, "comments": ["新用户文案"]}
  ]
}
```

仇恨值参考：`0.8+` 草了12次以上，`0.4` 草了6次左右，`0.0` 新用户。

## 概率机制说明

### 草群友触发

使用操作系统熵源（`secrets` 模块）生成真随机数判定，不可预测、不可复现。

### 注入量

采用**对数正态分布**，日常集中在几十到几百 mL，极限上限 100L。射完后附带评级：

| 评级 | 范围 |
|------|------|
| SSS | ≥ 4000 mL |
| S   | 800 - 3999 mL |
| A   | 300 - 799 mL |
| B   | 80 - 299 mL |
| C   | 20 - 79 mL |
| D   | < 20 mL |

评级字符可在 `main.py` 的 `_ml_grade()` 函数中自定义。

日常草nj中位数约200mL，≥5L概率约0.06%。反草险逃+老仇人时有概率进入「情绪失控」模式，中位数约830mL，≥5L约12%，≥10L约5%，100L极难但存在。

### 反草触发

草群友失败后，反草概率由以下四个因子动态计算，**不是固定概率**：

**Q — 逃脱质量（主导）**
用 sigmoid 函数将"逃走百分比"转换为逃脱质量值。逃走百分比越低（差点没跑掉），Q 越低，反草概率越高；跑得越干净，Q 越高，反草概率趋近于零。

**G — 累积仇恨**
该用户近30天草群友成功次数越多，G 越大。轻松逃走时仇恨被大幅压制，险逃时仇恨才会叠加发力——草的次数越多，越容易被针对。

**T — 今日群内压力**
当天群内已被草成功的总次数越多，T 越大，群友反扑意志越强。

**E — 随机熵扰动**
每次判定注入一个真随机扰动（±6%），让同一场景每次结果略有浮动，无法被预测。

`fancao_probability` 配置作为整体缩放系数，数值越高四个因子权重越大，上限随配置线性开放（设50时险逃+老仇人约70%-85%，设100时可接近98%）。

典型场景概率参考（`fancao_probability=50`）：

| 场景 | 反草概率 |
|------|---------|
| 险逃 + 新用户 | 约47% |
| 险逃 + 老仇人（草过15次+） | 75% - 85% |
| 刚好逃 + 普通用户 | 12% - 25% |
| 轻松逃 + 任何人 | <5% |

### 重置草nj

普通用户每天有 `reset_daily_attempts`（默认3）次尝试机会，每次按草群友触发概率掷骰，成功才重置，机会用完当天不能再试。bot管理员不受限制，可直接重置自己、指定人或全群。

## 数据说明

- **群友体内数据**：每月1日0点自动重置
- **草群友/反草/杂鱼战绩数据**：滚动统计近30天记录
- 数据存储在 AstrBot `data` 目录下，不会因插件更新丢失
